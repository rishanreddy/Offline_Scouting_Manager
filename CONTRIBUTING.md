# Contributing Guidelines

Thanks for contributing to Offline Scouting Manager.

## How to report issues

1. Open a GitHub issue using the issue templates.
2. Include:
   - App version
   - OS/platform
   - Steps to reproduce
   - Expected vs actual behavior
   - Screenshots/log export when possible

## How to submit pull requests

1. Fork and create a feature branch.
2. Keep changes focused to one concern.
3. Run lint/build locally before opening PR.
4. Provide PR summary with:
   - What changed
   - Why it changed
   - How it was verified
5. Add screenshots for UI changes.

## Code of conduct

By participating, you agree to:
- Be respectful and constructive
- Assume good intent
- Provide feedback focused on code and outcomes, not people
- Keep collaboration inclusive and professional

Unacceptable behavior includes harassment, discrimination, and personal attacks.
